# Продвижение постов

:::warning Предупреждение об отключении
Функциональность этого сценария интегрирована в [Super Marketing](./14.super-marketing.md). Пожалуйста, используйте Super Marketing для более мощных и гибких возможностей продвижения постов.
:::

Функция продвижения постов позволяет увеличить вовлечённость за счёт массовых лайков, репостов, добавления в избранное и просмотров. Система стремится имитировать поведение реальных пользователей, чтобы снизить риск обнаружения; однако соблюдайте правила платформы и контролируйте частоту операций.

## Использование в Super Marketing

1. Откройте скрипт: выберите «Super Marketing».
2. Выберите тип источника данных «Список ссылок на посты».
3. Вставьте ссылки на посты массово или импортируйте из текстового файла; после импорта выберите «Добавить» или «Перезаписать».
4. Выберите стратегию использования данных:

    - Общий пул: все боты взаимодействуют с одними и теми же постами.

    - Одноразовое потребление: каждый бот взаимодействует с разными постами, чтобы каждый пост обрабатывался только одним аккаунтом.

5. В «Действия с постом — Повторы» задайте количество взаимодействий для каждого поста. При включённых комментариях рекомендуем установить 1, чтобы избежать дублирования.
6. В «Действия с постом» выберите типы взаимодействий: лайк, поделиться, репост, в избранное, комментарий, просмотр видео и т. д.

     Если выбран «Комментарий», настройте параметры комментариев следующим образом:

     - Предварительно подготовленные шаблоны комментариев: вставьте многострочные или многопараграфные шаблоны комментариев (каждая строка/параграф считается отдельным кандидатом).

     - Порядок использования шаблонов: выберите **Случайный** или **По порядку**.
         - Случайный: каждый раз случайным образом выбирается один шаблон.
         - По порядку: используйте шаблоны в порядке вставки/импорта (полезно для детерминированной дедупликации).

     - Вставка эмодзи: опционально можно включить автоматическую вставку эмодзи в комментарии для повышения естественности.

     - Генерация комментариев в реальном времени (опционально — ChatGPT API):
         - При включении скрипт может вызывать API типа ChatGPT для генерации контекстно-ориентированных уникальных комментариев на основе заголовка/описания поста. Это значительно снижает повторяемость комментариев и подходит, когда требуется много уникальных комментариев.
         - Примечания: для реального времени требуется действительный API ключ; учитывайте ограничения по скорости, стоимость и вопросы конфиденциальности (не отправляйте чувствительные данные сторонним сервисам).

     - Два распространённых способа обеспечить уникальность комментариев:
         1. Загрузить и по очереди использовать подготовленные шаблоны в режиме «По порядку» (каждый шаблон используется один раз до исчерпания).
         2. Использовать ChatGPT API для динамической генерации уникального комментария для каждого поста (генерация на основе содержания).

     - Рекомендация: протестируйте стратегию комментариев на небольшой выборке, оцените качество и поведение платформы; при использовании генерации в реальном времени отслеживайте использование API и расходы.

7. Настройте остальные параметры (параллельность, задержки, пул аккаунтов), проверьте настройки и запустите скрипт.

## Рекомендации и замечания

- Сначала протестируйте на небольшом количестве постов, чтобы убедиться, что настройки не вызывают срабатывания защиты платформы.
- Комментарии должны быть разнообразными и естественными; избегайте вставки одинакового текста.
- Установите разумные задержки и параллельность, чтобы имитировать поведение человека.
- При длительной работе отслеживайте состояние аккаунтов и логи задач, корректируйте стратегию при необходимости.
