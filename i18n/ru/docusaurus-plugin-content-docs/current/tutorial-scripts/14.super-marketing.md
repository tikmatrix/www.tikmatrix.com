# Суперпродвижение

:::info
Скрипт Суперпродвижения объединяет подписки/отписки, личные сообщения, взаимодействия с постами и комментарии в одну очередь заданий, идеально подходит для быстрого охвата потенциальных клиентов или повторного вовлечения старых подписчиков в рамках одной операции.
:::

## Порядок действий

1. В TikMatrix/IgMatrix выберите устройство(а) для выполнения скрипта (поддерживается множественный выбор).
2. Откройте `Скрипты` → `Суперпродвижение`.
3. Выберите **Источник данных** (Имена пользователей или Ссылки на посты), затем нажмите `Импорт txt файла`, чтобы загрузить соответствующий файл `.txt`, или вставьте многострочные данные напрямую; после импорта выберите "Добавить" или "Перезаписать".
4. Включите модули действий с пользователями, постами и комментариями по мере необходимости и настройте их согласно приведённым ниже инструкциям.
5. Установите ползунок **Интервал заданий** для определения случайной задержки между целями, затем нажмите `Запустить скрипт` для выполнения задания.

## Настройка

### Источник данных

- **Имена пользователей / Ссылки на посты**: определяет доступные функции. Подписка, отписка и личные сообщения работают только с именами пользователей; если нужно работать только с постами, можно использовать режим ссылок.
- **Способ импорта**: Нажмите кнопку `Импорт txt файла`, чтобы загрузить соответствующий файл `.txt`, или вставьте многострочные данные напрямую; после импорта выберите "Добавить" или "Перезаписать".

### Стратегия использования данных

- **Общий пул (Shared pool)**: Все bot-аккаунты берут задания из одного пула целей, подходит для сценариев, где несколько аккаунтов должны многократно взаимодействовать с одними и теми же целями (например, совместная раскрутка популярного поста).

- **Однократное потребление (Single-consumption)**: Каждый bot-аккаунт обрабатывает только разные цели, гарантируя, что каждая цель обрабатывается только одним аккаунтом один раз, подходит для сценариев, где нужно избежать повторного охвата (например, массовая рассылка личных сообщений или уникальные комментарии).

  - **Лимит потребления на задачу**: При использовании режима однократного потребления можно установить максимальное количество целей, которые будет обрабатывать каждый запуск задачи. Это предотвращает потребление всех данных первой партией заданий, гарантируя наличие целей для последующих ротаций аккаунтов. Установите 0 для неограниченного потребления (поведение по умолчанию).

Выбор стратегии влияет на распределение очереди и параллельное выполнение, выбирайте исходя из целей и соображений контроля рисков.

### Действия с пользователями

- **Подписаться на пользователей**: При входе на главную страницу профиля в первую очередь выполняется действие подписки.
- **Отписаться от пользователей**: Отменяет подписку; можно запускать отдельно или для очистки подписчиков после отправки сообщений; не рекомендуется включать одновременно с "Подпиской".
- **Пожаловаться на аккаунт**: Отправляет жалобу на аккаунт пользователя за мошенническое поведение. Эта функция использует фиксированный путь жалобы и не поддерживает пользовательские пути.

  :::warning Путь жалобы (фиксированный)
  Пожаловаться → Пожаловаться на аккаунт → Something else → Frauds and scams
  :::

  - Примечание: Эта функция использует предопределённый путь жалобы. Пользовательские причины или пути жалобы не поддерживаются.
  - Используйте ответственно и только в законных целях. Злоупотребление может привести к ограничениям аккаунта.

- **Отправить личное сообщение**: Случайным образом выбирается одно сообщение из многострочных шаблонов; при включении "Автоматически добавлять эмодзи" автоматически добавляются естественные эмодзи. Скрипт проверяет, разрешает ли получатель личные сообщения, и пропускает, если отправка невозможна.

  - Шаблоны сообщений (предварительно сгенерированные): В "Настройках сообщений" вставьте многострочные или многопараграфные шаблоны сообщений (каждая строка/параграф рассматривается как кандидат сообщения).

  - Порядок использования шаблонов: Выберите "Случайный (Random)" или "По порядку (Sequential)".

    - Случайный: Каждый раз случайным образом выбирается шаблон для отправки.

    - По порядку: Шаблоны чередуются в порядке вставки/импорта, подходит для воспроизводимых стратегий дедупликации.

  - Вставка эмодзи: Можно включить автоматическую вставку эмодзи в сообщения или комментарии для повышения естественности.

  - Генерация в реальном времени (опционально, ChatGPT или совместимый API):

    - При включении скрипт может вызвать настроенный вами endpoint генерации (заполните API Key / URL / модель / System Prompt) для генерации персонализированного текста на основе контекста цели.
    - Внимание: Укажите действительный API Key и обратите внимание на ограничения скорости, стоимость и соблюдение конфиденциальности (избегайте отправки конфиденциальных данных).

  - Стратегии уникальности: Распространённые способы обеспечения уникальности каждого сообщения/комментария:

      1. Использование режима "По порядку" для чередования предварительно сгенерированных шаблонов (шаблоны используются только один раз до исчерпания).
      2. Использование генерации в реальном времени (ChatGPT) для генерации уникального текста для каждой цели.

  - Перед использованием массовой рассылки личных сообщений обязательно протестируйте на небольшой партии и обратите внимание на требования конфиденциальности и соответствия нормам.

### Действия с постами

- **Количество обрабатываемых постов**: Определяет количество постов, которые будет обрабатывать каждый аккаунт (1–50), требуется только когда источник данных - "Имена пользователей".
- **Количество повторений**: Повторяет цикл взаимодействия с постами несколько раз без перезапуска скрипта.
- **Длительность просмотра**: Установите минимальное и максимальное время просмотра (0–300 секунд). Скрипт случайным образом выберет значение в этом диапазоне перед выполнением взаимодействий.
- **Взаимодействия с постами**: Можно отдельно включить лайки, избранное, репосты и шеринг. При включении нескольких действий система распределит их по различным временным точкам в течение периода просмотра для более естественного поведения.

- **Комментарии**: При включении можно оставлять комментарии к постам.

     Если включены "Комментарии", настройте следующие параметры в настройках комментариев:

  - Предварительно сгенерированные шаблоны комментариев: Вставьте многострочные или многопараграфные предварительно сгенерированные шаблоны комментариев (каждая строка или параграф рассматривается как кандидат комментария).

  - Порядок использования шаблонов: Выберите "Случайный" или "По порядку".
    - Случайный: Каждый раз случайным образом выбирается из кандидатов.
    - По порядку: Используются в порядке вставки/импорта (подходит для воспроизводимых стратегий дедупликации).

  - Вставка эмодзи: Можно включить автоматическую вставку эмодзи в комментарии для повышения естественности.

  - Генерация комментариев в реальном времени (опционально, ChatGPT API):
    - При включении скрипт может вызвать ChatGPT-подобные API для генерации контекстно-релевантных и уникальных текстов комментариев на основе заголовка/описания поста в реальном времени. Этот метод может значительно снизить риск дублирования комментариев, подходит для сценариев, требующих большого количества уникальных комментариев.
    - Примечание: Для использования генерации в реальном времени требуется действительный API Key, обратите внимание на ограничения скорости, стоимость и соблюдение конфиденциальности (не отправляйте конфиденциальные данные в сторонние сервисы).

  - Два распространённых способа обеспечения уникальных комментариев:
         1. Использование режима "По порядку" для загрузки и чередования предварительно сгенерированных шаблонов комментариев (гарантирует, что каждый шаблон используется только один раз до исчерпания).
         2. Использование ChatGPT API для генерации каждого комментария в реальном времени (динамически генерируется на основе заголовка/содержания поста, естественным образом легче обеспечить уникальность).

  - Рекомендация: Сначала протестируйте стратегию комментирования на небольшой партии постов, проверьте качество комментариев и обработку платформой; если используете генерацию в реальном времени, следите за объёмом вызовов API и затратами.

  - При использовании стратегии "По порядку" система комбинирует номера заданий для чередования шаблонов между несколькими устройствами, снижая вероятность дублирования; при использовании AI-генерации рекомендуется применять простые правила проверки качества (например, диапазон длины, запрет URL / чувствительных слов).

### Слияние задач (объединение по одинаковому имени пользователя)

- Описание: при включении опции «Слияние задач по одинаковому имени пользователя» несколько целей для одного и того же имени пользователя выполняются в рамках одной задачи (последовательно или параллельно в зависимости от реализации). Даже если одна цель завершится неудачей, вся задача не будет прервана, а оставшиеся цели продолжат выполнение, что снижает накладные расходы на переключение задач и планирование.

- Поведение при включении: несколько целей, направленных на один аккаунт, объединяются в единый блок задачи. Ошибки затрагивают только отдельную цель и фиксируются, но не блокируют продолжение других целей. Режим подходит для масштабных маркетинговых запусков и может существенно снизить накладные расходы на запуск/переключение задач.

- Поведение при отключении: каждая цель создаётся как отдельная задача; любая ошибка немедленно регистрируется и останавливает соответствующую подзадачу (не влияя на другие независимые задачи). Режим полезен, когда требуется точный мониторинг по каждой цели и оперативное оповещение.

- Рекомендации:
  - Включайте этот режим при необходимости высокой пропускной способности и больших пакетах задач, чтобы снизить расходы на планирование.
  - Отключайте его, если требуется точный статус выполнения по каждой цели и немедленные действия при ошибках (например, ручное вмешательство или откат).

### Интервал заданий

- Установите минимальное и максимальное значения (в минутах). Скрипт будет случайным образом ждать в пределах диапазона между каждой целью, плюс собственные короткие паузы.

## Процесс выполнения

- Если включён режим множественных аккаунтов, TikMatrix/IgMatrix сначала запустит TikTok/Instagram и переключится на соответствующий аккаунт.
- В режиме имён пользователей система открывает каждую главную страницу профиля выбранным способом доступа, ждёт завершения загрузки, затем последовательно выполняет действия подписки, отправки сообщений, отписки; если получатель блокирует личные сообщения, автоматически пропускает.
- Взаимодействия с постами будут прокручивать страницы профиля или открывать ссылки напрямую, просматривать согласно заданному количеству и длительности, и выполнять действия лайка, избранного, репоста, шеринга, пропуская рекламу, прямые эфиры и уже лайкнутый контент.
- При использовании ссылок на посты скрипт будет открывать ссылки одну за другой, выполнять настроенные действия и повторять цикл согласно "Количеству повторений".
- Модуль комментариев будет брать текст из шаблонов или вывода AI. Режим "По порядку" будет комбинировать номера заданий для распределения разных комментариев между несколькими устройствами.
- Между каждой целью скрипт добавляет небольшие случайные паузы, затем добавляет случайное ожидание из интервала заданий для имитации человеческого ритма.

## Лучшие практики

- Сначала проверьте текст и темп на небольших партиях, затем масштабируйте после подтверждения отсутствия проблем.
- Комбинирование нескольких действий (например, подписка + личное сообщение + лайк) может улучшить сигналы вовлечённости аккаунта.
- Держите личные сообщения и комментарии разговорными и искренними, чтобы избежать срабатывания модерации платформы.
- Регулярно обновляйте библиотеки комментариев или настраивайте AI System Prompt, чтобы сделать контент более релевантным темам кампании.
- Следите за ограничениями скорости платформы; для больших объёмов чередуйте аккаунты или прокси-узлы.
