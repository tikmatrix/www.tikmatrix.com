# Массовые личные сообщения

:::warning
Эта функция объединена в [Super Marketing скрипт](./14.super-marketing.md). Используйте Super Marketing для более мощных и гибких возможностей массовой рассылки и управления сообщениями.
:::

Массовые личные сообщения (Mass DM) используются для установления контакта путём отправки личных сообщений в большом объёме — подходит для продвижения, обратной связи с клиентами и реактивации пользователей. Система поддерживает заранее подготовленные шаблоны и генерацию сообщений в реальном времени, а также пытается имитировать человеческий ритм отправки, чтобы снизить риск детекции.

## Использование в Super Marketing

1. Откройте скрипт Super Marketing.
2. Выберите тип источника данных: «Список аккаунтов (целевой список)».
3. Вставьте список имён пользователей или импортируйте `.txt` файл (по одному имени в строке). После импорта можно выбрать "Добавить" (Append) или "Заменить" (Replace).
4. Выберите стратегию использования данных:

   - Общий пул: все боты берут цели из одного общего списка.

   - Однократное потребление: каждый бот обрабатывает уникальные цели, так что каждая цель получает сообщение только от одного аккаунта.

5. Установите, сколько сообщений отправлять на одну цель (обычно 1, чтобы не спамить).
6. Настройте параметры сообщения:

   - Предварительно подготовленные шаблоны: вставьте несколько строк или абзацев готовых текстов. Каждая строка/абзац считается отдельным шаблоном.

   - Порядок использования: выберите «Случайный» или «По порядку».

   - Случайный: при отправке выбирается случайный шаблон.

   - По порядку: шаблоны используются в порядке вставки/импорта (удобно при необходимости воспроизводимой стратегии дедупликации).

   - Вставка эмодзи: включите автоматическое добавление эмодзи для большей естественности.

   - Генерация сообщений в реальном времени (опционально, ChatGPT API):

      - При включении скрипт может вызывать ChatGPT или похожий API для генерации персонализированных сообщений на основе имени пользователя или доступного контекста — полезно, если нужно много уникальных сообщений.

      - Внимание: для работы требуется валидный API-ключ; учитывайте ограничения по скорости запросов, стоимость и требования по конфиденциальности (не отправляйте чувствительные данные).

   - Два распространённых способа обеспечить уникальность сообщений:

   1. Ротация предварительно подготовленных шаблонов по порядку (каждый шаблон используется один раз до исчерпания).

   2. Генерация уникальных сообщений в реальном времени через ChatGPT.

7. Выберите способ открытия профиля пользователя (если требуется просмотреть профиль): "Поиск" или "Прямой переход".
8. Настройте интервал задач (мин/макс в минутах), параллельность и пул аккаунтов, затем запустите скрипт.

:::tip Об интервалах задач
Задайте минимальный и максимальный интервал и включите случайные задержки. Разнесённые по времени действия снижают риск синхронных пиков и ограничений со стороны платформы.
:::

## Рекомендации и примечания

- Делайте тексты сообщений разнообразными и соответствующими правилам платформы; избегайте массовой отправки одинаковых сообщений.
- При использовании генерации в реальном времени следите за расходом API и стоимостью, а также не передавайте конфиденциальную информацию.
- Если часть целей не обработана, проверьте логи и повторно запустите обработку — уже обработанные строки будут пропущены.
