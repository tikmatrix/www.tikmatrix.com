# Продвижение пользователей

:::warning Предупреждение об отключении
Функциональность этого сценария интегрирована в [Super Marketing](./14.super-marketing.md). Пожалуйста, используйте Super Marketing для более мощных и гибких возможностей продвижения пользователей.
:::

Функция продвижения пользователей позволяет увеличить охват и вовлечённость за счёт массовой подписки или отписки на целевые аккаунты. Система стремится имитировать поведение реальных пользователей, чтобы снизить риск обнаружения; однако соблюдайте правила платформы и контролируйте частоту операций.

## Использование в Super Marketing

1. Откройте скрипт: выберите «Super Marketing».
2. Выберите тип источника данных «Список аккаунтов пользователей».
3. Вставьте имена пользователей массово или импортируйте из текстового файла (по одному имени в строке); после импорта выберите «Добавить» или «Перезаписать».
4. Выберите стратегию использования данных:

   - Общий пул: все боты взаимодействуют с одними и теми же целями.

   - Одноразовое потребление: каждый бот обрабатывает разные цели, чтобы каждая цель обрабатывалась только одним аккаунтом.

5. В разделе «Действия с пользователем — Повторы» задайте количество повторов для каждой цели (обычно 1).
6. Выберите тип операции: Подписка или Отписка. Дополнительно в разделе «Действия с пользователем» можно настроить:

    - Метод перехода в профиль: выберите **Поиск пользователя** или **Прямое открытие профиля (Direct Open Profile)**.

    - Отправка личного сообщения (опционально): при включении вставьте заранее подготовленные многострочные шаблоны сообщений (каждая строка или абзац — один шаблон). Настройте порядок использования шаблонов:

          - **Случайный**: каждый раз выбирается случайный шаблон.
          - **По порядку**: шаблоны используются в порядке вставки/импорта (могут продолжаться между запусками задач).

       Также можно включить автоматическую вставку эмодзи в шаблоны для большей естественности.

7. Настройте остальные параметры (параллельность, задержки, пул аккаунтов), проверьте настройки и запустите скрипт.

## Рекомендации и замечания

- Сначала протестируйте на небольшом числе целей, чтобы убедиться, что настройки не вызывают срабатывания защиты платформы.
- Устанавливайте разумные задержки и уровень параллельности, чтобы имитировать поведение человека.
- При длительных запусках отслеживайте состояние аккаунтов и логи задач и корректируйте стратегию по необходимости.
- Операции по продвижению пользователей не гарантируют 100% успеха; при неудачах повторите задачу или проверьте статус целевого аккаунта.
