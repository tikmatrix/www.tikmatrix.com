# Super Boost

:::info
Super Boost объединяет подписки/отписки, личные сообщения, взаимодействия с постами и комментарии в одной очереди заданий. Подойдёт, когда нужно быстро охватить потенциальную аудиторию или оживить старых подписчиков.
:::

## Порядок запуска

1. Выберите устройство(а) в TikMatrix/IgMatrix для выполнения сценария.
2. Откройте `Scripts` → `Super Boost`.
3. Выберите **Data source** (`Usernames` или `Post links`). Нажмите `Select` или вставьте строки напрямую. После импорта можно выбрать `Append` или `Overwrite`.
4. Включите блоки User Actions, Post Actions и Comments по необходимости и настройте их ниже.
5. Установите ползунок **Task interval** для задания случайной задержки между целями и нажмите `Start Script`.

## Источники данных

- **Usernames / Post links**: определяет доступные действия. Подписки/отписки и личные сообщения требуют Usernames; операции только с постами работают по ссылкам.
- **Импорт**: используйте `Select` или `Import txt` для загрузки UTF-8 `.txt` файла или вставки многострочных данных. После импорта выберите `Append` или `Overwrite`.

## Стратегия использования данных

- **Shared pool**: все аккаунты берут цели из общего пула. Подходит, когда нужно, чтобы несколько аккаунтов взаимодействовали с одними и теми же популярными целями (например, раскрутка трендового поста).

- **Single-consumption**: каждая учётная запись обрабатывает уникальные цели, чтобы избежать повторных обращений к одному и тому же объекту; полезно для массовой рассылки DM или единственного комментария на цель.

Выбор стратегии влияет на распределение очереди и параллельность — подбирайте под цель кампании и риск-менеджмент.

## Действия с профилем

- **Follow users**: выполняет подписку при заходе на профиль.
- **Unfollow users**: удаляет подписку; можно запускать отдельно или после рассылки сообщений. Не рекомендуется включать Follow и Unfollow одновременно.
- **Send DM**: скрипт выбирает одну строку из шаблонов сообщений; опция `Insert emoji` добавляет эмодзи. Сообщения не отправляются пользователям, которые их блокируют.

- Шаблоны DM (предгенерированные): вставьте многострочные шаблоны в Message Settings (каждая строка/параграф – отдельный вариант).

- Порядок использования шаблонов: Random или Sequential.

  - Random: случайный выбор шаблона.

  - Sequential: последовательное использование шаблонов для предсказуемой дедупликации.

- Insert emoji: автоматическое добавление эмодзи к сообщениям/комментариям.

- Insert device id (опционально): добавление идентификатора устройства для трекинга (только для отладки/аудита; не рекомендуется в проде).

- Генерация в реальном времени (опционально, совместимо с ChatGPT API):

  - При включении скрипт вызывает настроенный endpoint (заполните API Key / URL / модель / System Prompt) для генерации персонализированного текста по контексту цели.

  - Внимание: предоставьте рабочий API Key, следите за лимитами и расходами, не отправляйте чувствительные данные.

- Стратегии уникальности:

  1. Чередовать предгенеренные шаблоны в режиме Sequential (каждый шаблон используется один раз до исчерпания).

  2. Использовать генерацию в реальном времени (ChatGPT) для уникального текста на каждую цель.

- Перед масштабированием протестируйте небольшую партию DM и учитывайте требования приватности и compliance.

## Действия с постами

- **Max posts to process**: число постов на аккаунт/в цикле (1–50). Требуется при использовании Usernames.
- **Repeat times**: количество повторов цикла взаимодействий без перезапуска сценария.
- **View duration**: мин/макс время просмотра (0–300 с). Скрипт случайно выбирает значение в диапазоне перед выполнением действий.
- **Post interactions**: лайк, добавление в избранное, репост и шаринг; при включении нескольких действий они распределяются по времени просмотра.

## Комментарии

- **Mass comments**: доступно в режимах Usernames и Post links.
- **Manual comments**: по одному комментарию на строку; выберите `Random` или `Sequential`. Sequential распределяет комментарии по очереди заданий, снижая повторяемость между устройствами.
- **AI comments**: включите `Generate by ChatGPT`, заполните URL, API Key, модель и System Prompt, затем нажмите `Test`. TikMatrix отправит подпись поста модели и опубликует сгенерированный ответ.
- **Insert emoji**: добавляет случайные эмодзи к комментариям из шаблонов или AI.

При использовании Sequential режим комбинирует номера заданий и порядок шаблонов для ротации между устройствами; при AI-генерации применяйте простые проверки качества (длина, отсутствие URL/запрещённых слов).

## Интервал заданий

- Установите мин/макс (в минутах). Скрипт ждёт случайное значение в диапазоне плюс короткие внутренние паузы. Для возврата к настройкам по умолчанию используйте `Reset settings`.

## Исполнение

- Если указаны Usernames, TikMatrix запустит TikTok и переключится на соответствующие аккаунты.
- В режиме Usernames скрипт открывает профиль выбранным способом, ждёт загрузки и выполняет Follow, DM, Unfollow по очереди; DM пропускаются для пользователей с блокировкой сообщений.
- Взаимодействия с постами пролистывают/открывают контент, смотрят ролики заданное время и выполняют выбранные действия, пропуская рекламу, прямые эфиры и уже залайканные публикации.
- В режиме Post links Super Boost открывает каждую ссылку и выполняет настройки, повторяя цикл согласно Repeat times.
- Комментарии берутся из шаблонов или от AI; Sequential режим распределяет комментарии между устройствами.
- Между целями скрипт вставляет короткие случайные паузы и выдерживает Task interval для имитации человека.

## Рекомендации

- Тестируйте на небольших объёмах перед масштабированием.
- Комбинируйте действия (follow + DM + like) для усиления вовлечённости.
- Держите сообщения и комментарии естественными, чтобы снизить риск блокировок.
- Обновляйте библиотеку комментариев и адаптируйте System Prompt под кампанию.
- Следите за лимитами платформы и переключайте аккаунты/прокси при больших объёмах.
