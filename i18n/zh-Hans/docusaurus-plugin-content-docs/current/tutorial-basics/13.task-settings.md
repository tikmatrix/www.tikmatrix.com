# 任务设置

## 任务调度规则

- 任务在到达开始时间后 10 秒内会被执行。
- 每台设备同时只会运行 1 个任务，其他任务会在该设备上自动排队等待处理。
- 最大并发任务数受许可等级限制；免费用户不会运行任何自动化任务，会一直显示为“等待中”。
- 每次重试前系统会添加 5 到 60 秒的随机延迟。

该对话框用于控制任务的重试策略与执行顺序。下面简要说明每个选项及使用方法。

## 自动重试

作用：

- 自动重试失败的任务，直到成功或达到重试上限。

如何使用：

- 当失败多为短暂故障（网络波动、应用异常）时启用。
- 先在小范围任务上验证行为，再决定是否全量启用。

## 最大重试次数

作用：

- 设置任务自动重试的最大尝试次数。

如何使用：

- 建议初始设置为 2–3，若遇到大量短暂失败可适当上调。
- 界面会限制为合理范围（例如 1–10）。

## 随机顺序

作用：

- 在每台设备上以随机顺序执行任务，避免同步行为。

如何使用：

- 在大规模设备池或希望降低可检测性时启用。

## 生效与提示

- 修改后通常会立即生效；界面会显示保存成功或失败的通知。

## 在哪里打开

- 打开设备页或任务页，点击任务设置（齿轮）图标。

## 快速建议

- 从 `最大重试次数` = 2–3 开始调整。
- 大规模运行时启用 `随机顺序` 可减少同步模式。
- 在小范围先测试 `自动重试` 行为。
