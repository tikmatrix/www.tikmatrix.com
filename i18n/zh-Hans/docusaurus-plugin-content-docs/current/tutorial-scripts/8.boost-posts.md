# 提升帖子

:::warning 下线提示
该脚本功能已整合到 [超级营销脚本](./14.super-marketing.md)。请优先使用“超级营销”脚本以获得更强大、更灵活的帖子提升能力。
:::

提升帖子功能可通过批量点赞、分享、收藏、转发与观看等操作来增加帖子的互动量。系统会尽量模拟真实用户行为以降低被检测的风险，但使用时仍应遵循平台规则并适度控制频率。

## 在“超级营销”脚本中使用提升帖子功能

1. 打开脚本：选择“超级营销”。
2. 选择数据源类型为“帖子链接列表”。
3. 批量粘贴帖子链接或从文本文件导入；导入后可选择“追加”或“覆盖”当前列表。
4. 选择数据使用策略：

    - 共享池策略：所有 bot 账号对相同帖子进行互动。

    - 单次消耗策略：每个 bot 账号只对不同的帖子进行一次互动，确保每个帖子仅被一个账号处理一次。

5. 在“帖子操作 — 重复次数”中设置每个帖子的互动次数。若包含评论，建议将评论次数设置为 1，避免出现重复评论。
6. 在“帖子操作”中选择需要的互动类型：点赞、分享、转发、收藏、评论、观看视频等。

     如果选择“评论”，请在评论设置中进行以下配置：

     - 预生成评论模板：可粘贴多行或多段预生成评论文案（每行或每段视为一条候选评论）。

     - 文案使用顺序：选择“随机”或“按顺序”使用。
         - 随机：每次从候选文案中随机抽取一条。
         - 按顺序：按粘贴/导入的顺序依次使用（适合需要可复现的去重策略）。

     - 插入表情（emoji）：可开启自动在评论中插入表情以提高自然度。

     - 实时生成评论（可选，ChatGPT API）：
         - 启用后，脚本可调用 ChatGPT 类 API，根据帖子标题/描述实时生成上下文相关且唯一的评论文案。此方式可显著降低重复评论的风险，适合需要大量唯一评论的场景。
         - 注意事项：使用实时生成需提供可用的 API Key，并注意速率限制、费用与隐私合规（不要发送敏感数据到第三方服务）。

     - 保证唯一评论的两种常见方式：
         1. 使用“按顺序”加载并轮换预生成评论模版（保证每条模版只被使用一次直到耗尽）。
         2. 使用 ChatGPT API 实时生成每条评论（基于帖文标题/内容动态生成，天然更易保证唯一性）。

     - 建议：先用小批量帖子测试评论策略，检查评论质量与平台处理情况；若使用实时生成，请监控 API 调用量与成本。
7. 配置其他脚本参数（如并发数、延时、账号池等），确认无误后启动脚本。

## 使用建议与注意事项

- 建议先在少量帖子上进行测试，确认策略与参数不会触发平台风控。
- 评论内容应多样化并保持自然，避免复制粘贴相同文本。
- 合理设置延时与并发，模拟人工行为以降低风险。
- 长期运行时请定期检查账号健康与任务日志，并根据情况调整策略。
