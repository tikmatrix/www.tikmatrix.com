# 批量私信

:::warning 下线提示
该功能已整合到 [超级营销脚本](./14.super-marketing.md)。请优先使用“超级营销”脚本以获得更强大、更灵活的批量私信与消息管理能力。
:::

批量私信功能用于通过批量发送私信（DM）与目标用户建立联系，适用于推广、客服回访和用户唤醒等场景。系统支持预生成模板与实时生成两种消息来源，并会尽量模拟人工发送以降低被检测风险。

## 在“超级营销”脚本中使用批量私信功能

1. 打开脚本：选择“超级营销”。
2. 选择数据源类型为“用户账号列表（目标名单）”。
3. 批量粘贴目标用户名或从文本文件导入（每行一个）；导入后可选择“追加”或“覆盖”。
4. 选择数据使用策略：

    - 共享池策略：所有 bot 账号对相同目标名单进行私信。

    - 单次消耗策略：每个 bot 账号只处理不同的目标，确保每个目标仅由一个账号发送消息一次。

5. 设置每个目标的消息发送次数（通常为 1 次，以避免重复骚扰）。
6. 在“消息设置”中配置私信内容与发送策略：

    - 预生成消息模板：可粘贴多行或多段预生成私信文案（每行或每段视为一条候选消息）。

    - 文案使用顺序：选择“随机”或“按顺序”使用。

        - 随机：每次随机抽取一条模板发送。

        - 按顺序：按照粘贴/导入顺序轮换使用模板（适合需要可复现的去重策略）。

    - 插入表情（emoji）：可开启自动在消息中插入表情以提高自然度。

    - 实时生成消息（可选，ChatGPT API）：

        - 启用后，脚本可调用 ChatGPT 或类似 API，根据目标用户名或可用上下文实时生成个性化消息，适合需要大量唯一消息的场景。

        - 注意：启用实时生成需提供有效 API Key，并注意速率限制、费用与隐私合规（避免发送敏感数据）。

    - 保证唯一消息的两种常见方式：

        1. 使用“按顺序”轮换预生成模板（每条模板只使用一次直到耗尽）。

        2. 使用 ChatGPT 实时生成每条消息（基于目标信息动态生成，唯一性更高）。

7. 配置打开用户方式（如需查看用户资料）：选择“搜索用户”或“直接打开主页（Direct Open Profile）”。
8. 配置任务间隔（最小/最大，单位：分钟）与并发、账号池等参数，确认无误后启动脚本。

:::tip 关于任务间隔
建议设置最小/最大间隔并启用随机间隔，以避免所有设备同时执行导致异常峰值，从而降低被平台检测或限制的风险。
:::

## 使用建议与注意事项

- 私信内容应合规且多样化，避免短时间内发送大量相同内容。
- 如使用实时生成，请严格监控 API 调用量与费用，并确保不向外泄露敏感信息。
- 任务可能失败；遇到失败请重试或查看日志，系统会自动跳过已完成的目标。
